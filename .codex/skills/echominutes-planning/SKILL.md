# EchoMinutes Planning

Use this skill to keep EchoMinutes-Agent work aligned with the product manual and delivery slices.

## Workflow

1. Read `AGENTS.md` and `docs/EchoMinutes-Agent.md`.
2. Identify whether the request belongs to P0, P1, P2, P3, or is out of first-version scope.
3. Keep the deliverable narrow enough to verify.
4. Prefer mock-provider workflows until the provider interfaces are stable.
5. Call out any request that implies login, SaaS admin, payment, online collaboration, cloud sync, or real-time recording.

## Delivery Map

Load `references/delivery-map.md` when a task needs a more detailed slice breakdown.

## Planning Output

When asked for a plan, include:

- active delivery slice
- files or modules likely affected
- implementation steps
- validation command or manual check
- explicit non-goals
